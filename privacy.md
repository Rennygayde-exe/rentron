## 1. What RenTron Collects
RenTron only collects and stores data that’s necessary for operation, such as:

- **User IDs and Message IDs:** Used internally for application systems, ticket management, and moderation tools.  
- **Server Configuration Data:** Includes channel IDs and data prune schedules.

RenTron **does not** store chat messages, private messages, analytics data, or the contents of any user conversations.

---

## 2. How RenTron Uses Data
RenTron uses collected data solely to operate its core features and ensure reliable performance.  
This includes:

- Managing applications and ticket systems (e.g., application status, open or closed tickets).  
- Performing scheduled maintenance tasks such as automated data pruning.

RenTron does **not** use user data for marketing, advertising, analytics, or profiling purposes.  
All information remains securely hosted and managed within RenTron’s controlled environment in Seattle, Washington.

---

## 3. Data Retention and Removal
All stored data is retained **only as long as necessary** to provide its intended function. For example:

- Application and ticket records are deleted after their purpose is complete.  
- Maintenance logs (such as pruning data) are periodically cleared automatically.

---

## 4. Data Sharing and Disclosure
RenTron does **not** share, sell, or trade user data with any third parties.  
Access is limited to:

- The bot owner and authorized maintainers.

---

## 5. Security Measures
RenTron employs appropriate technical safeguards to protect user data, including:

- Local and independent storage of all configuration and user data.  
- Encrypted access to storage drives.  
- Avoiding unnecessary data collection and minimizing data retention.

While no system is entirely immune to security risks, RenTron is designed to manage user data responsibly and transparently.  
RenTron’s source code is open for review at [GitHub](https://github.com/Rennygayde-exe/rentron).
